## 4. 剩余风险

### 4.1 合约层面（前端无法消除）

| 风险 | 状态 | 说明 |
|-----|------|------|
| **第三方审计缺失** | ❌ 未完成 | V2 合约未经独立审计。FundVaultV01、LockRewardManagerV02、BeneficiaryModuleV02 均为自研合约，存在未知漏洞风险。上线前强烈建议至少完成 1 次快速安全审查（Sherlock / Code4rena / Cantina）。 |
| **externalTransfersEnabled 默认 false** | ⚠️ 需手动操作 | 合约部署后 `externalTransfersEnabled = false`，`transferToStrategyManager()` 会 revert。管理员必须在上线前手动调用启用。Pre-launch Verification 面板已标注此项，但不会自动修复。 |
| **lockLedger 零地址** | ⚠️ 需确认 | 若 FundVaultV01.lockLedger() 返回零地址，`claimExitAssets` 不会扣减锁仓份额，emergency exit 时已锁仓用户可能被多计配额。Pre-launch 面板已标注。 |
| **reserveRatioBps 默认 10000 (100%)** | ⚠️ 需手动设置 | 部署后默认值意味着策略投资被完全阻断。管理员必须在投入运营前设置合理值（建议 2000–3500 bps）。Dashboard 已显示警告。 |
| **mgmtFeeBpsPerMonth 需配置** | ⚠️ 需手动设置 | 若保持 0，协议无收入。Pre-launch Verification 面板已标注。 |

### 4.2 Aave V3 APY 数据

| 风险 | 说明 |
|-----|------|
| **APY 公式精度** | 当前使用线性近似：`rate / 1e27 * 100`。Aave 官方使用复利公式 `(1 + rate/SECONDS_PER_YEAR/1e27)^SECONDS_PER_YEAR - 1`。偏差约 0.01%–0.1%，对 UI 展示影响可接受，但不应作为精确财务计算依据。 |
| **getReserveData ABI tuple 字段顺序** | Aave V3 在不同链上的 Pool 实现略有差异。若 Base mainnet 的 tuple 字段顺序与 ABI 定义不符，`currentLiquidityRate` 读取可能为 0 或错误值，导致 APY 显示为 "—"。上线后应在真实环境验证一次读数。 |

### 4.3 PPS Sparkline 数据质量

| 风险 | 说明 |
|-----|------|
| **localStorage 仅本地** | 每个用户设备独立积累，无法展示协议历史走势。新用户始终看到 "Accumulating data…"。这是设计约束，非 bug，但会影响首次用户体验。 |
| **1 小时采样间隔** | 数据积累缓慢，需约 30 小时才能看到完整 sparkline。早期用户数据点稀少。 |

### 4.5 前端已知局限

| 局限 | 说明 |
|-----|------|
| 无 gasless / meta-tx 支持 | 用户必须持有 ETH 支付 gas，门槛略高 |
| Governance 页 createProposal 仅限 PROPOSER_ROLE | 普通用户无法发起提案，治理实质上仍为中心化 |
| Emergency Exit 后无自动通知机制 | 用户不会收到任何链外通知，只有主动访问 /vault 才能看到 Emergency Exit 面板 |
| claimExitAssets：用户需手动输入 burn 数量 | 无法一键全额 claim（MAX 按钮已提供，但需要用户操作） |
